# تطبيق شركة العنقاء للمقاولات — Flutter + Firebase

تطبيق حقيقي لإدارة طلبات تصنيع المطابخ، يعمل على iOS و Android من نفس الكود،
مربوط بقاعدة بيانات Firebase حقيقية بحيث تكون الطلبات مشتركة وفورية بين
كل الأجهزة (العميل والمدير).

## الميزات

- **تسجيل دخول إلزامي برقم الجوال** (رمز تحقق SMS عبر Firebase Auth) — لا أحد
  يدخل التطبيق بدون تسجيل دخول.
- **حماية زر المدير**: زر "مدير" ما يظهر إلا لأرقام الجوال المدرجة يدويًا في
  قائمة `admins` داخل Firestore. أي شخص ثاني ما يقدر يوصل للوحة التحكم حتى
  لو عرف عن التطبيق، لأن الحماية على مستوى الخادم (Firestore Rules) مو بس
  على مستوى الواجهة.
- **الطلبات تنحفظ عند المدير مباشرة** عبر Firestore (قاعدة بيانات سحابية) —
  أي طلب يرسله عميل يظهر فورًا عند المدير من أي جهاز.
- **العميل يشوف حالة طلبه والمدة المتوقعة** من نفس تبويب "طلباتي": الحالة
  (قيد المراجعة / قيد التصنيع / تم التسليم / مرفوض)، السعر بعد ما يرفعه
  المدير، وعدد الأيام المتوقعة للتسليم (وتاريخ متوقع محسوب تلقائيًا).
- **تنزيل صورة المنتج** أو **تنزيل تقرير PDF كامل بالطلب** (يشمل كل
  التفاصيل + الصورة) من داخل بطاقة الطلب، عبر قائمة المشاركة/الحفظ في
  الجهاز.
- لوحة تحكم المدير: فلترة وبحث، تغيير الحالة، تحديد السعر، تحديد المدة
  المتوقعة، تتبع دفع العربون والمبلغ الكامل، وجدول أسعار الخامات المرجعي.

## 1) قبل التشغيل — أكمل إعداد Firebase

المشروع مربوط مسبقًا بمشروعك `alanqaa-cd82d` (ملفا `google-services.json`
و `GoogleService-Info.plist` موجودون فعليًا، وملفات Gradle معدَّلة). يبقى
عليك 3 أشياء من طرف Firebase Console:

### أ) إضافة رقم/أرقام المدراء
من [Firebase Console](https://console.firebase.google.com) → مشروعك →
**Firestore Database** → أنشئ مجموعة (Collection) باسم بالضبط:
```
admins
```
وأضف داخلها **مستند (Document)** لكل رقم مدير، بحيث يكون **معرّف المستند
نفسه (Document ID)** هو رقم الجوال بصيغة دولية كاملة، مثال:
```
+966501234567
```
(الحقول داخل المستند مو مهمة، يكفي وجود المستند بهذا الاسم). أي رقم مو
موجود هنا ما رح يقدر يشوف زر "مدير" ولا يوصل للوحة التحكم حتى لو حاول.

### ب) نشر قواعد الأمان (Security Rules)
الملفان `firestore.rules` و `storage.rules` الموجودان بجذر المشروع جاهزين،
لكن لازم تنشرهم يدويًا من الـ Console:
- **Firestore Database → Rules**: انسخ محتوى `firestore.rules` والصقه ثم
  اضغط **Publish**.
- **Storage → Rules**: انسخ محتوى `storage.rules` والصقه ثم اضغط **Publish**.

بدون نشر هذي القواعد، القواعد الافتراضية إما تمنع كل شي أو تسمح بكل شي حسب
وضع الإنشاء (Test/Production mode).

### ج) تفعيل Phone Authentication (إذا ما فعّلته)
**Authentication → Sign-in method → Phone → تفعيل**. أثناء التطوير، Firebase
يسمح بعدد محدود من الرسائل مجانًا يوميًا وقد يطلب منك أيضًا إضافة أرقام
اختبار (**Phone numbers for testing**) لتفادي التحقق الحقيقي أثناء التجربة.

## 2) التشغيل محليًا

```bash
flutter pub get
flutter run
```

> ملاحظة: لا حاجة لتشغيل `flutterfire configure` — التطبيق يهيّئ Firebase
> تلقائيًا من ملفي `google-services.json` و `GoogleService-Info.plist`
> الموجودين مسبقًا في المشروع (`main.dart` يستخدم `Firebase.initializeApp()`
> بدون خيارات يدوية).

## 3) بناء نسخة للنشر

```bash
# Android (APK)
flutter build apk --release

# Android (App Bundle لمتجر Google Play)
flutter build appbundle --release

# iOS (يحتاج Mac + Xcode + حساب مطور Apple)
flutter build ios --release
```

## كيف تعمل الحماية بالتفصيل

1. أي شخص يفتح التطبيق **لازم** يسجّل دخول برقم جواله (رمز SMS) قبل ما يشوف
   أي شاشة — هذا يغطي طلبك بجعل الدخول برقم الجوال إلزاميًا.
2. بعد الدخول، التطبيق يسأل Firestore: "هل رقم هذا المستخدم موجود في مجموعة
   `admins`؟" إذا الجواب نعم، يظهر له زر التبديل "عميل / مدير" في الأعلى.
   إذا لا، ما يشوف الزر أصلاً ويبقى بواجهة العميل فقط.
3. حتى لو حاول شخص متمرّس يتلاعب بالتطبيق نفسه ليظهر له الزر، **قواعد
   Firestore نفسها** (`firestore.rules`) ترفض أي تعديل على حالة الطلب أو
   السعر أو المدة إلا إذا كان رقمه فعليًا مسجّل في `admins` — يعني الحماية
   حقيقية على مستوى الخادم مو بس شكلية بالواجهة.

## ملاحظات مهمة

- **رقم فهرس (Index) قد يُطلب من Firestore تلقائيًا**: أول مرة تفتح تبويب
  "طلباتي"، قد يظهر خطأ برسالة تحتوي رابط لإنشاء فهرس مركّب (composite
  index) لأن الاستعلام يفلتر برقم الجوال ويرتّب بالتاريخ. افتح الرابط
  واضغط "إنشاء" (يأخذ دقيقة أو دقيقتين) ثم أعد المحاولة — مرة وحدة بس.
- **تنزيل PDF** يستخدم خطوط عربية (Noto Naskh Arabic) تُحمَّل تلقائيًا من
  Google Fonts أول مرة يولّد فيها المستخدم ملف PDF على جهازه (يحتاج
  إنترنت في تلك اللحظة فقط، ثم تُخزَّن مؤقتًا).
- إن واجهت مشاكل بناء على iOS بخصوص الحد الأدنى للإصدار، ارفع
  `IPHONEOS_DEPLOYMENT_TARGET` في Xcode إلى 15.0 (المشروع حاليًا على 13.0).
